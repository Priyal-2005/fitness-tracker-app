# Priyal's Fitness — PWA Install Guide

This app is a fully offline-capable PWA.
After one build, it runs on your phone without any server, laptop, or internet.

---

## ✅ What's already set up

| Feature | Status |
|---|---|
| Service worker (offline caching) | ✅ vite-plugin-pwa / Workbox |
| Web App Manifest | ✅ inside vite.config.js |
| Dark splash screen | ✅ index.html inline style |
| iOS Add to Home Screen | ✅ apple-mobile-web-app meta tags |
| Android Install prompt | ✅ manifest + icons |
| Standalone mode (no browser chrome) | ✅ display: standalone |
| localStorage data (offline) | ✅ no backend needed |

---

## 🚀 Step 1 — Build the app

Run this once on your laptop:

```bash
cd priyal-fitness
npm install          # first time only
npm run build        # creates the dist/ folder
```

The `dist/` folder is your complete app — all assets, service worker, and manifest included.

---

## 📱 Step 2 — Install on your Android phone

### Option A: Local network (fastest for testing)

1. Run the preview server:
   ```bash
   npm run preview
   ```
   It starts at `http://localhost:4173`

2. Find your laptop's IP address:
   - **Windows:** Open Command Prompt → type `ipconfig` → look for `IPv4 Address` (e.g. `192.168.1.42`)
   - **Mac:** Open Terminal → type `ipconfig getifaddr en0` or `en1`
   - **Linux:** `hostname -I`

3. Make sure your phone and laptop are on the **same Wi-Fi network**

4. Open **Chrome** on your Android phone (must be Chrome, not Samsung Browser)

5. Go to: `http://192.168.1.42:4173` ← replace with your actual IP

6. Wait for the page to fully load (first load caches everything)

7. Tap the **3-dot menu (⋮)** in Chrome → tap **"Add to Home Screen"** or **"Install App"**

8. Confirm → app icon appears on your home screen

9. Open it — it now runs in full-screen standalone mode, no browser bar

---

### Option B: Netlify (permanent, best option — free)

Deploy once, access from anywhere, works offline after first load.

1. Build the app:
   ```bash
   npm run build
   ```

2. Go to: **https://app.netlify.com**

3. Sign up / log in (free)

4. Click **"Add new site"** → **"Deploy manually"**

5. Drag the entire `dist/` folder into the upload box

6. Netlify gives you a URL like: `https://priyal-fitness-abc123.netlify.app`

7. Open that URL on your phone in Chrome

8. Tap **3-dot menu → "Add to Home Screen"**

9. Done. Works offline forever after first load.

> **Tip:** Every time you update the app, run `npm run build` again and re-drag the `dist/` folder to Netlify. Your localStorage data is never affected by updates.

---

### Option C: iOS Safari (iPhone)

1. Open Safari on your iPhone (must be Safari, not Chrome)
2. Go to your Netlify URL (Option B) or local IP (Option A)
3. Tap the **Share button** (the box with an arrow pointing up)
4. Scroll down → tap **"Add to Home Screen"**
5. Name it "Fitness" → tap Add

> Note: iOS Safari does not show an "install" prompt like Android. The above steps are the only way.

## 🔌 How offline works

After the first load, the service worker (built by Workbox) caches:
- All JavaScript bundles
- CSS files
- HTML
- Icons and images

From that point, the app opens and runs with **zero internet connection**.
Your data (workouts, weight, supplements) lives in localStorage on your device
and is never affected by going offline.

---

## 🔄 How to update the app

1. Make your code changes
2. Run `npm run build` again
3. If using Netlify: re-drag `dist/` to Netlify
4. Open the app on your phone — it auto-updates in the background (registerType: 'autoUpdate')
5. Your data is never lost during updates

## 🗂 What the dist/ folder contains

After `npm run build`:

├── manifest.webmanifest          ← auto-generated by vite-plugin-pwa
├── sw.js                         ← service worker (Workbox)
├── workbox-*.js                  ← Workbox runtime

├── icon-512.png
└── assets/

Everything is self-contained. No server needed to run it.

## 🐛 Troubleshooting

**"Add to Home Screen" not appearing on Android:**
- Make sure you're using Chrome (not Samsung Browser or Firefox)
- Make sure the site loaded over HTTPS (Netlify) or localhost/local IP
- Wait ~30 seconds on the page — Chrome needs to verify the PWA criteria

**App shows blank screen after install:**
- Open Chrome → 3-dot menu → Settings → Site settings → find your site → Clear & reset
- Re-open and let it fully load again

**Old version showing after update:**
- Close the app completely (remove from recents)
- Re-open — the service worker auto-updates in the background

**localStorage data missing:**
- Data is stored per-origin (URL). If you switch from local IP to Netlify, data starts fresh.
- Once you pick a permanent URL (Netlify), always use that — data persists forever.

## ⚡ Performance notes

- First load: ~200–400ms (all assets from cache after first visit)
- Subsequent loads: instant (service worker intercepts and serves from cache)
- localStorage operations: synchronous, zero network cost
- Total bundle size: typically ~150–250kb gzipped for this app
